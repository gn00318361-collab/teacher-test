# 青溪國小代理代課教師甄試管理系統 v1.0

![Language](https://img.shields.io/badge/Language-HTML%20%2F%20CSS%20%2F%20JavaScript-blue)
![Storage](https://img.shields.io/badge/Storage-localStorage-orange)
![Deployment](https://img.shields.io/badge/Deployment-GitHub%20Pages-green)

一套專為學校行政設計的資料驅動型試務管理系統。本專案旨在打破傳統校務行政高度依賴分散 Word 文件的痛點，將「事前準備、甄試當天、甄試後」的複雜流程、人員編組、評審委員與試務文件，整合進單一網頁控制台，實現「單一來源維護，標準文件輸出」的數位轉型目標。

## 專案核心目標

- 系統化管理：建立可重複使用的管理模組，而非僅製作單次性的資訊展示網頁。
- 響應式連動：人員、委員、工作清單 Checklist 可即時維護，異動時全案數據自動響應。
- 經驗反哺閉環：整合試務日誌，將當日突發狀況與改善建議一鍵轉為明年優化任務，實現系統自我升級。
- 模組化延伸：保持底層架構彈性，未來可延伸至資優營隊、招生、成果展等其他校務專案。

## 七大階段開發成果

### 1. 頂部試務控制 Dashboard

- 考程階段自動判定：系統動態讀取時間，自動比對並顯示目前所屬的試務階段。
- 四大核心 KPI 看板：即時統計目前考程階段、全案未完成工作數、待指派或待確認人數、工作完成率進度條。
- 數據即時響應：下游任何資料新增、修改、刪除或勾選後，Dashboard 數據立即重新渲染。

### 2. 人員管理模組

- 動態人力指派：支援工作人員新增、修改、刪除，並可指定負責日期與職務。
- 尚未安排提醒：系統會依日期檢查核心工作是否缺人，協助試務承辦快速補位。

### 3. 評審委員管理模組

- 日期分類瀏覽：自動依據委員可支援日期分流呈現。
- 狀態追蹤標籤：動態標示已確認、待確認、部分時間，方便人事窗口追蹤進度。

### 4. 智慧 Checklist 管理模組

- 多維度控管：支援任務新增、修改、刪除，可設定行政階段、優先順序與完成期限。
- 高效檢索：內建即時關鍵字搜尋與排序功能。
- 年度初始化：提供還原預設資料功能，方便跨年度重新建立專案。

### 5. 文件中心與 Word 一鍵整合

- HTML-to-Word Blob 技術：不依賴後端伺服器與複雜套件，利用純前端技術將結構化資料一鍵打包成 `.doc` 表格文件。
- 實體成果輸出：支援一鍵匯出人員分工表與委員評分表，讓行政核定、列印與會議使用更順手。

### 6. 試務日誌與反哺機制

- 標籤化問題紀錄：提供分類選單，記錄當日問題、狀況描述與明年建議。
- 系統自我升級：每筆日誌旁設有「轉為明年優化工作」按鈕，點擊後自動新增 `【明年優化】` 高優先度任務，讓今年經驗成為明年防錯提醒。

### 7. 靜態部署與單機可用

- 無登入、無資料庫：適合校內行政快速使用，也方便在單機環境保存資料。
- GitHub Pages 友善：可直接部署為靜態網站，提供團隊共同檢視。

## 技術架構與開發原則

- 前端技術：原生 `HTML5`、`CSS3`、`JavaScript`，無重度框架依賴，確保輕量與快速載入。
- 資料儲存：資料保存於瀏覽器 `localStorage`，包含專案資料、Checklist 同步資料與試務日誌。
- 核心儲存鍵：
  - `qxes-subteacher-system-v2`：主要專案資料。
  - `cx_checklist`：Checklist 同步資料。
  - `cx_daily_logs`：試務日誌資料。
- 零後端維護：無須配置資料庫與伺服器路由，適合校園內網環境或 GitHub Pages 快速部署。

## 檔案結構

```text
├── index.html                         # 單一網頁主結構
├── styles.css                         # 介面視覺與排版樣式
├── app.js                             # 核心邏輯、localStorage 狀態管理、DOM 渲染、Word 匯出
├── 試務廣播稿範本.docx                # 已產生之 Word 文件
├── 成績登記評審簽章表範本.docx        # 已產生之 Word 文件
└── README.md                          # 專案說明文件
```

## 使用與部署說明

1. 將本專案 Clone 或下載至本機電腦。
2. 直接開啟 `index.html` 即可在瀏覽器中使用完整功能。
3. 若要提供校內團隊共同檢視進度，可將此專案推至 GitHub 儲存庫，並開啟 GitHub Pages。

## 適用場景

- 代理代課教師甄試。
- 校內行政專案控管。
- 招生、營隊、成果展等需分工、流程與文件輸出的活動。
- 年度例行業務的經驗累積與改進追蹤。
